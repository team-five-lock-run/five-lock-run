# 🚂 오락실행
이 프로젝트는 **명절 기간 수많은 사용자가 동시에 몰리는 상황**을 가정하여, 안정적인 열차 예매 서비스를 제공하기 위한 시스템입니다.
좌석 예약은 **Redis 기반의 분산 락 구조**로 처리하고, 예매 전후 흐름과 대기열 시스템까지 포함하여 실시간성과 무결성을 함께 달성하는 것을 목표로 합니다.

### ✅ 핵심 기능

1. **열차 관리 시스템**

  * 역, 노선, 열차, 차량, 좌석, 스케줄을 등록하고 관리
  * 다양한 예매 케이스를 커버할 수 있도록 유연한 구조 설계

2. **좌석 예약 시스템 (Redis 기반)**

  * 실시간 좌석 예약 정보는 Redis에 저장
  * **Redisson의 분산 락**으로 하나의 좌석에 단 1명만 예약 가능
  * 예약과 예매를 분리해 성능 최적화와 데이터 무결성 확보

3. **대기열 시스템**

  * 동시 접근 시 진입 순서를 제어하여 서버 안정성 확보
  * 실시간 순번 조회, 대기열 이탈, 예약 완료 시 자동 이탈 처리

## ⚙️ 주요 기능

### 👤 사용자

* 회원가입 / 로그인 / 로그아웃
* 마이페이지 조회 및 정보 수정
* 회원 탈퇴

### 🛤️ 열차 운영

* 역 / 노선 / 열차 / 스케줄 등록 및 활성화/비활성화
* 열차 차량 및 좌석 구성 수정
* 프리미엄칸 / 일반칸 좌석 조회

### 💸 가격 정책

* 가격 등록 / 수정 / 삭제
* 좌석 등급별 가격 목록 및 단건 조회

### 📦 예약 (Redis 기반)

* 좌석 예약 등록 (Redisson 락 기반)
* 사용자 예약 목록 조회
* 좌석별 예약 목록 조회
* 예약 단건 조회 / 예약 취소
* Redis 성능 최적화: SCAN 기반 목록 조회

### 🎟️ 예매 (DB 저장)

* 결제 후 예매 등록
* 사용자 예매 목록 및 단건 조회
* 예매 취소
* 예매 정보를 기반으로 티켓 발급

### ⏳ 대기열 시스템

* 대기열 입장 / 순번 조회 / 이탈
* 예약 완료 시 자동 이탈 처리
* 실시간 알림 구조 설계
